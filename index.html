WHAT THIS TOOL DOES

从库存预测，走到可以执行的补货决定

这个作品不是简单套公式。它把周度判断、未来库存、物流时间和箱规放进同一条计算链，最后给出可以直接执行的下单建议。

CALCULATION LOGIC

四步得到补货建议

规则保持透明，每个输出都能回到原始日期和公式核对。

当前示例计算轨迹LIVE
判断周待计算 → 预警待计算 → 断货待计算 → 入库待计算
  1. A
    预计入库日期

    周一 + 生产交期 + 入库上架天数。

  2. B
    断货预警日期

    下一次预计断货日向前推该产品填写的预警天数。

  3. 判
    本周下单判断

    A晚于或等于B所在周周日，则需要下单。

  4. E
    整箱补货数量

    覆盖天数 × 平均日销，再按箱规向上取整。

GUARDRAILS

数据不完整时，不强行给答案

补货建议会影响真实采购，所以系统先判断数据是否可信，再决定能不能继续计算。

输入校验日期连续库存周期预测范围
↗未来数据不够

明确告诉你预测到哪天、还缺到哪天。

⟳当前已经缺货

区分临时缺货、后续恢复和持续缺货。

⊘日期有问题

不是星期一、重复或断档都会停止计算。
