  • 预测范围必须覆盖计算区间

Auditable logic

每一个结果都有可追溯的业务依据

01

预计入库

A = 判断周周一 + 交期 + 入库上架天数

02

断货预警

B = 下一轮预计断货日期 − 固定21天

03

下单判断

A晚于或等于B所在周的周日，则本周需要下单。

04

整箱补货

C = A − B + 14；E = C × 平均日销，并按箱规向上取整。

Reliability by design

不为了给答案而编造答案

供应链决策最危险的不是空白，而是看起来精确的错误结果。

预测范围保护

没有足够未来数据时，结果保持“待确认”，不会把“没找到断货”误判成“无需下单”。

库存周期识别

判断周已经缺货但未来会入库时，跳过当前缺货段，从恢复后的新周期寻找下一次断货。

日期完整性

检查星期一、连续日期、重复日期、跨年与闰年，避免一天偏差改变下单结论。

异常停止机制

持续缺货、预警逾期、控制销量缺失等情况停止标准算量，并明确提示人工介入。
